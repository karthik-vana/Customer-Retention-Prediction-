Data Processing: Input customer data is cleaned, encoded, and scaled using the best-performing techniques identified during model training.

â“ FAQ

What is the primary evaluation metric?

ROC-AUC (Area Under the Receiver Operating Characteristic Curve) is the primary metric because it's robust to class imbalance and measures the model's ability to distinguish between classes across all thresholds.

How is the best technique selected in each stage?

Each stage evaluates 5 different techniques and selects the one with the best performance on the validation set using appropriate metrics (cross-validation ROC-AUC, test-set accuracy, etc.)

Can I use this model for my telecom company?

Yes! The model is production-ready and can be deployed using the provided Flask app. You can retrain it with your own dataset following the same pipeline.

How accurate is the prediction?

The accuracy depends on the input data quality and the selected model. Check the model comparison charts and ROC curves in the overview section for detailed metrics.

What if I need to add new features?

You can extend the feature engineering module to add new feature extraction techniques or modify the feature list in the FEATURE_DEFINITIONS dictionary in app.py.
